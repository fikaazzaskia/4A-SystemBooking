4.4 Pengujian User — Booking

IDSkenarioInputHasil Diharapkan
U-01 Booking normal berhasil Data lengkap, slot kosong, tanggal tidak diblokir HTTP 200, kode_booking dikembalikan (format BRB-XXXXXX)
U-02 Booking data tidak lengkap Tanpa field nama HTTP 422, "Data tidak lengkap"
U-03 Booking di tanggal yang diblokir Tanggal yang ada di tabel blocked_dates HTTP 422, "Tanggal ini tidak tersedia untuk booking"
U-04 Booking slot yang sudah terisi Barber + waktu yang sudah ada booking pending/confirmed HTTP 409, "Slot waktu ini sudah dipesan, silakan pilih waktu lain"
U-05 Double booking oleh 2 user bersamaan (race condition) 2 request booking ke slot yang sama hampir bersamaan Salah satu berhasil, satu lagi mendapat HTTP 409
U-06 Booking di hari libur (jadwal.libur = 1) Pilih tanggal Minggu (libur) Frontend tidak menampilkan slot (validasi jadwal)
